# Figma 화면설계서 제작 요청 프롬프트

## 프로젝트 개요
"리드로그(READLOG)" 독서 소셜 앱의 화면설계서(와이어프레임)를 제작해줘. 모바일 앱(iOS/Android 공용) 기준이고, 아래 요구사항과 사용자 플로우에 맞는 전체 화면 세트를 만들어줘.

## 디자인 톤
- 독서/책 테마에 맞는 차분하고 따뜻한 톤 (베이지, 우드톤, 딥그린 계열 추천)
- 미니멀한 카드형 UI, 정보 밀도가 높지 않게
- 모바일 우선, 기준 디바이스는 iPhone 15(390x844)로 설계하되 절대 픽셀값에 고정하지 말고 Flexbox 기반의 반응형 레이아웃으로 설계해줘 (React Native 개발 시 다양한 화면 크기에 자동 대응 가능해야 함)

## 반응형 설계 시 추가 고려사항
- **기준 비율**: iPhone 표준 비율(9:19.5) 기준으로 작업하되, 실제 구현은 디바이스별 자동 비율 조정이 이루어지므로 절대 사이즈보다 레이아웃 구조와 비율에 집중해줘
- **가변 콘텐츠 제약**: 카드 그리드(내 서재), 진도 바(멤버별 진도 현황판), 댓글 리스트처럼 콘텐츠 양이 가변적인 요소는 최소/최대 너비 기준을 함께 명시해줘 (작은 화면에서 레이아웃이 깨지지 않도록)
- **Safe Area**: 상단 네비게이션 바와 하단 탭바는 노치·홈 인디케이터를 고려한 Safe Area 안쪽에 배치된다고 가정하고 디자인해줘

## 화면 목록 및 요구사항

### 1. 회원관리
- **로그인 화면**: ID/PW 입력 필드, 로그인 버튼, 회원가입 이동 링크, 에러 메시지 노출 영역
- **회원가입 화면**: ID(중복확인 버튼 포함)/PW/PW확인/닉네임 입력, 가입 완료 버튼
- **마이페이지**: 프로필(닉네임, 프로필 이미지), 독서기록·한줄평·댓글·독서그룹 활동 탭 또는 리스트, 설정 진입 (북마크 탭은 독서기록과 방향성이 겹쳐 제외 — 2026-07-06)
- **정보 수정 화면**: 닉네임/프로필 이미지 수정
- **비밀번호 변경 화면**: 기존 PW 확인 → 신규 PW 입력
- **회원탈퇴 화면**: PW 확인 → 탈퇴 경고 안내 → 확인 버튼

### 2. 독서 플랜 관리
- **책 검색 화면**: 검색창, 알라딘 API 기반 검색 결과 리스트(표지+제목 카드형)
- **책 상세/등록 화면**: 표지, 제목, "내 서재에 추가" 버튼, 독서 상태 선택(읽는 중/완독)
- **내 서재 화면**: 등록한 책 목록(표지 그리드), 완독 권수 집계 배너/이미지
- **한줄평 작성 화면**: 완독한 책에 대한 한 문장 코멘트 입력/수정/삭제
- **SNS 공유 화면**: 사용자가 업로드한 사진 + 이모지 스티커 추가 + 코멘트 입력 → Instagram 공유 버튼

### 3. 같이 책 읽기
- **독서모임 목록 화면**: 모임 카드 리스트(모임명, 인원, 진행 도서), 모임 개설 버튼
- **모임 개설 화면**: 모임명, 공개/비공개 토글, 최대 인원 설정, 독서 기간(시작~종료일) 선택, 도서 선택
- **모임 참가 화면**: 초대 코드/링크 입력 필드
- **모임 홈 화면**: 모임 정보, 멤버 리스트, 진행 도서, 진도 현황판(멤버별 진도 바), 댓글/설정 탭 이동
- **초대 화면**: 초대 링크 복사, 초대 코드 표시, 공유 버튼
- **진도 공유 화면**: 진도 북마크 추가, 챕터/위치 입력
- **공유 책 댓글 화면**: 댓글 리스트(스포일러 블러 처리 포함), 댓글 입력창, 이모지 반응, 원문 인용 버튼
- **모임 설정 화면**: 공개여부, 인원, 기간, 챕터별 목표 수정, 멤버 권한 관리(강퇴, 위임)

## 화면 흐름 (각 화면을 아래 흐름에 맞게 연결해줘)
1. 앱 시작 → 로그인 상태 분기 → (비로그인 시) 로그인/회원가입 → 메인 홈
2. 메인 홈 → 독서 플랜 / 같이 책 읽기 / 마이페이지 3개 탭
3. 독서 플랜: 책 검색 → 등록 → 독서 상태 설정 → (완독 시) 한줄평 → SNS 공유
4. 같이 책 읽기: 모임 목록 → 개설 또는 참가 → 모임 홈 → 진도공유/댓글/설정

## 요청 사항
- 각 화면은 프레임 단위로 분리하고, 화면 간 연결을 Figma의 프로토타입 커넥터(화살표)로 표시해줘
- 우선순위 "상" 기능(회원가입, 로그인, 독서 상태, 완독 권수, SNS공유, 모임 개설/삭제, 멤버 초대, 공유소설 진도공유, 스포일러 방지)은 상세 컴포넌트까지 구성해줘
- 우선순위 "중/하" 기능은 와이어프레임 수준의 레이아웃만 잡아줘도 괜찮아
- 공통 컴포넌트(상단 네비게이션 바, 하단 탭바, 버튼, 입력 필드)는 별도 컴포넌트로 만들어 재사용해줘
