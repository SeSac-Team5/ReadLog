# 2026-07-02 작업 로그

## 통합 테스트 시 고려 사항 — `_shared` ↔ 모듈 SKILL.md 정합성

### 배경

`_shared/SKILL.md`와 `reading-group-module/SKILL.md`를 교차 분석한 결과,  
모듈 SKILL.md에 `_shared` 규칙이 명시되지 않은 항목들이 확인됨.  
SKILL.md 파일 자체는 수정하지 않으며, 통합 테스트 단계에서 아래 항목들을 체크리스트로 활용할 것.

> 동일한 구조적 이슈는 `auth-module`, `reading-plan-module`에도 동일하게 적용될 가능성이 높음.

---

### 체크리스트

#### 1. 디자인 토큰 하드코딩 여부
- `#2D4A3E`, `#FDFBF4`, `#EDE7D8` 등 색상값이 각 모듈 컴포넌트에 직접 문자열로 입력되어 있는지 확인
- **기준**: 모든 색상 참조는 `frontend/src/constants/theme.ts` 상수 import로만 이루어져야 함
- **대상 경로**: `frontend/src/components/reading-group/**`, `frontend/src/screens/reading-group/**`

#### 2. 공용 컴포넌트 중복 생성 여부
- `NavBar`, `TabBar`, `PrimaryBtn`, `Field` 등을 모듈 폴더 내에 별도로 재구현했는지 확인
- **기준**: `frontend/src/components/common/`의 공용 컴포넌트만 사용, 모듈 자체 구현 금지
- **주의**: TabBar는 4탭 고정 (홈/내 서재/독서모임/마이), 좌측 슬라이드 방식은 폐기됨

#### 3. axios 인스턴스 자체 생성 여부
- `frontend/src/api/reading-group/` 내부에 새로운 axios 인스턴스나 독립 fetch 래퍼를 생성했는지 확인
- **기준**: `frontend/src/api/client`의 공용 인스턴스를 import해서 엔드포인트 함수만 추가해야 함

#### 4. Backend 공용 레이어 직접 수정 여부
- `backend/app/core/`, `backend/app/db/`, `backend/app/common/` 파일이 각 모듈 PR의 diff에 포함되어 있는지 확인
- **기준**: 해당 경로는 읽기 전용으로 import만 허용, 수정 시 이슈 등록 후 3인 합의 필요

#### 5. 모듈 간 단방향 의존성 위반 여부
- `reading-group` 화면/스토어 코드에서 `screens/home` 또는 `store/home` 코드를 직접 import했는지 확인
- **기준**: 홈 → 각 모듈(읽기 전용 호출) 방향만 허용, 역방향 참조 금지
