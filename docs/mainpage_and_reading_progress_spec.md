# 메인페이지 추가 — 디자인 스펙

> 대상 화면: 메인 홈 (`MainHomeScreen`)
> 근거 문서: `requirements_and_flowchart.md` 1-4. 메인페이지

---

## 1. 화면 목적

로그인 후 진입하는 첫 화면. 사용자의 독서 현황과 참여 중인 모임 진도를 한눈에 보여주고, 각 상세 화면(내 서재 / 독서모임 목록)으로 빠르게 진입시키는 대시보드형 홈.

---

## 2. 레이아웃 구조

```
┌───────────────────────────────┐
│ 상단 헤더                       │  READLOG 로고 · 알림 벨 아이콘
├───────────────────────────────┤
│ 독서 현황 카드                  │  딥그린 배경, 모서리 둥근 카드
│  - 완독 권수 (대형 숫자)          │
│  - 읽는 중인 책 표지 + 진행률 바   │
│  - [더보기] → 내 서재            │
├───────────────────────────────┤
│ 모임 진도 미리보기                │  섹션 타이틀 + [더보기] → 독서모임 목록
│  - 참여 모임 0개 → 빈 상태        │  일러스트(이모지) + CTA 버튼
│  - 참여 모임 1개 이상 → 카드 목록  │  모임별 멤버 진도 바
├───────────────────────────────┤
│ 하단 탭바                       │  홈 / 내 서재 / 독서모임 / 마이 (active: 홈)
└───────────────────────────────┘
```

- 기준 디바이스: iPhone 15 (390×844), Flexbox 기반 반응형 — 절대 px 고정 금지
- Safe Area: 상단 헤더·하단 탭바는 노치/홈 인디케이터 안쪽 배치

---

## 3. 기능 명세

| 기능 | 내용 | 중요도 |
|---|---|---|
| 독서 현황 카드 | 완독 권수 + 읽는 중 책 표지·진행률 바 노출 | 상 |
| 모임 진도 미리보기 | 참여 모임별 멤버 진도 바 목록 | 상 |
| 빈 상태(Empty State) | 참여 모임 0개 시 일러스트(이모지) + CTA 버튼 노출 | 상 |
| 더보기 버튼 | 독서 현황 카드 → 내 서재 / 모임 진도 미리보기 → 독서모임 목록 진입 | 중 |
| 상단 헤더 | READLOG 로고 + 알림 벨 아이콘 (홈 전용, 공용 NavBar 미사용) | - |
| 하단 탭바 | 4탭 구조(홈/내 서재/독서모임/마이) 공용 컴포넌트, `active="home"` | - |

---

## 4. 상태 분기

- **모임 진도 미리보기**
  - 참여 모임 `0개` → 빈 상태 UI (일러스트 + "모임 찾아보기" 등 CTA)
  - 참여 모임 `1개 이상` → 모임별 카드 리스트, 멤버 진도 바 노출
- **멤버 진도 바 색상**
  - 100% 완독 → 초록색 강조
  - 진행 중 → 딥그린(브랜드 컬러)
  - (기존 `GroupHomeScreen`과 동일한 패턴 유지)

## 5. 디자인 토큰

| 용도 | 값 |
|---|---|
| 카드 배경 (딥그린) | `#2D4A3E` |
| 텍스트/배경 (베이지, 밝은 톤) | `#FDFBF4` |
| 배경 (베이지, 톤다운) | `#EDE7D8` |
| 완독 진도 강조 | Green 계열 (예: `green-500`) |

- 색상은 기존 앱 팔레트(딥그린/베이지 계열)와 일관성 유지
- 카드형 UI, 정보 밀도는 낮게 — 미니멀 톤 유지

---

## 6. 확정된 결정 사항

- 좌측 슬라이드 메뉴(하이브리드 구조) **폐기** → 하단 탭바 단일 구조로 확정
- "전체보기" 버튼 명칭 → **"더보기"** 로 확정
- 독서 상태는 읽는 중 / 완독 2단계만 사용 (읽고 싶다 단계 없음)

## 7. 미결 사항

- 그룹 도서 진도 단위 (페이지 vs %) 확정 필요 → 진도 바 표시 방식에 영향
- 빈 상태 CTA 버튼의 정확한 문구/도착 화면(모임 목록 vs 모임 개설) 미확정

# 독서플랜 화면 추가 설계
- **(신규)독서 진도 입력 화면**: "내 서재 화면"에서 "읽는 중"인 책을 선택해 진입. 구성 요소는 다음과 같음
  - 책 표지/제목 헤더
  - 진도 입력 방식: 전체 페이지 수 대비 현재 페이지(숫자 입력) 또는 진행률 슬라이더(%) 중 선택 가능
  - 입력된 진도를 즉시 시각화하는 진행 바(가변 너비, 최소/최대 폭 기준 명시)
  - 진도 기록 일시 자동 저장, 이전 진도 기록 리스트(타임라인 형태, 선택사항)
  - "저장" 버튼 → 저장 시 내 서재 화면의 진도율이 갱신됨
  - 우선순위: "상" (상세 컴포넌트까지 구성)
- **SNS 공유 화면**: 사용자가 업로드한 사진 + 이모지 스티커 추가 + 코멘트 입력 + **진도 시각화 요소(신규)** + Instagram 공유 버튼
  - **(신규) 진도 시각화 요소**: 독서 진도 입력 화면에서 저장된 진도 데이터를 가져와 사진 레이아웃 위에 오버레이로 표시
    - 표시 형태(택1 또는 혼합 제공): ① 원형 게이지(%) 스티커, ② 가로 진행 바 스티커, ③ "p.120/350 읽음" 텍스트 배지
    - 사용자가 오버레이의 위치/크기를 자유롭게 조정(드래그) 가능
    - 오버레이 노출 여부 토글(끄기 가능)
    - 완독 시에는 진도 대신 "완독" 뱃지로 자동 전환되는 옵션 고려
  - 우선순위: "상" (상세 컴포넌트까지 구성)
